## 🎯 发布步骤

### 方法1：在微信开发者工具中发布（推荐）

#### 步骤1：上传代码
1. 打开微信开发者工具
2. 确保项目已导入并编译成功
3. 点击右上角 **"上传"** 按钮
4. 填写版本号：`1.0.39`
5. 填写项目备注：
   ```
   - 恢复云开发配置
   - 修复导入问题
   - 更新版本到1.0.39
   ```
6. 点击"确定"上传

#### 步骤2：登录微信公众平台
1. 访问：https://mp.weixin.qq.com
2. 使用微信扫码登录
3. 进入你的小程序管理后台

#### 步骤3：提交审核
1. 点击左侧 **"版本管理"**
2. 找到刚上传的版本 `1.0.39`
3. 点击 **"提交审核"**
4. 填写审核信息：
   - 版本号：`1.0.39`
   - 备注：`修复云开发配置，恢复项目正常运行`
5. 上传测试账号（如果需要）
6. 选择测试选项
7. 提交等待审核

#### 步骤4：审核通过后发布
1. 收到审核通过通知
2. 进入 **"版本管理"**
3. 点击 **"发布"** 按钮
4. 确认发布信息
5. 点击"确定"发布

---

### 方法2：使用 CLI 命令发布

#### 前提条件
1. 已开启 CLI 服务端口
2. 微信开发者工具已打开项目

#### 发布命令

**1. 上传代码**
```powershell
& "C:\Program Files (x86)\Tencent\微信web开发者工具\cli.bat" --upload "c:/Users/nierhan/CodeBuddy/Claw/travel-buddy/client" "1.0.39" "@version"
```

**参数说明：**
- `--upload`：上传小程序代码
- 路径：client 目录路径
- 版本号：`1.0.39`
- `@version`：使用 Git 版本号作为备注

#### 发布步骤
1. 在 PowerShell 中执行上述命令
2. 等待上传完成
3. 登录微信公众平台
4. 提交审核
5. 审核通过后发布

## ⚠️ 注意事项

### 发布要求
1. 必须通过微信认证
2. 必须有正式 AppID
3. 必须配置合法域名
4. 必须通过内容审核

### 审核时间
- 通常 1-7 个工作日
- 节假日可能延长
- 可以加急审核（需要付费）

### 发布限制
- 每天最多发布 10 次
- 发布后无法回退
- 体验版只能体验 30 天

## 🚨 回滚方案

如果发布后出现严重问题，可以：

### 紧急回滚
1. 立即下线小程序
2. 修复问题
3. 重新上传新版本
4. 重新提交审核

### 版本回退
1. 保留上一个版本
2. 不发布新版本
3. 等待问题修复后再发布
