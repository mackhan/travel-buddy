# 修复说明 - 云开发连接错误

## 问题描述
小程序启动时出现云开发 SDK 初始化错误，无法连接后端服务。

## 错误原因
1. 小程序尝试通过微信云托管（wx.cloud.callContainer）连接后端
2. 云托管配置指向云端服务，而非本地服务器
3. 本地开发环境没有配置云托管，导致连接失败

## 修复方案

### 1. 移除云开发依赖
- 从 app.js 移除 `wx.cloud.init()` 调用
- 从 app.json 移除 `"cloud": true` 配置
- 从 project.config.json 移除 `cloudfunctionRoot` 配置
- 移除 cloudEnvId 和 cloudServiceName 配置

### 2. 改为直接连接本地服务器
- 使用 `wx.request()` 直接请求 http://localhost:3000
- 移除云托管的 callContainer 调用
- 保持 baseUrl 配置指向本地服务器

### 3. 保留自动降级机制
- 后端连接失败时自动切换到 Mock 模式
- Mock 模式使用模拟数据，不影响功能体验

## 修改文件

### client/app.js
```javascript
// 移除前
wx.cloud.init({
  env: this.globalData.cloudEnvId,
  traceUser: false
})

wx.cloud.callContainer({
  config: { env: this.globalData.cloudEnvId },
  path: '/api/health',
  ...
})

// 修改后
wx.request({
  url: this.globalData.baseUrl + '/health',
  method: 'GET',
  ...
})
```

### client/app.json
```json
// 移除前
{
  "cloud": true
}

// 修改后
{
  // 已移除 cloud 配置
}
```

### client/project.config.json
```json
// 移除前
{
  "cloudfunctionRoot": "cloudfunctions/"
}

// 修改后
{
  // 已移除 cloudfunctionRoot 配置
}
```

2. 观察控制台输出
3. 应该看到：`✅ 后端服务已连接`

### 3. 验证功能
- 首页显示旅行列表
- 导航栏可以切换页面
- 所有页面正常加载

## 预期结果

### 成功连接
- 控制台显示：`✅ 后端服务已连接`
- 弹出提示：`✅ 服务器已连接`
- mockMode: false（使用真实后端）

### Mock 模式（备用）
- 控制台显示：`🎭 Mock 模式已启用`
- 弹出提示：`⚠️ 无法连接服务器，使用演示数据`
- mockMode: true（使用模拟数据）

两种模式都可以正常使用所有功能！
